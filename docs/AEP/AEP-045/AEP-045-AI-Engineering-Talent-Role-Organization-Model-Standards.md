# AEP-045 — AI Engineering Talent, Role & Organization Model Standards

**Version:** 1.0 (Draft)  
**Status:** Reference Standard

---

# 1. Purpose

Dokumen ini menetapkan standar role, capability, operating model, dan struktur organisasi untuk menjalankan AI Software Factory.

Tujuan utama:

* Mendefinisikan peran manusia dalam organisasi AI Engineering.
* Menentukan kompetensi yang dibutuhkan.
* Mengatur kolaborasi manusia dan AI Agent.
* Membentuk operating model yang scalable.
* Menjadi dasar pembentukan tim AI-native engineering.

AI Software Factory membutuhkan organisasi baru, bukan sekadar tim software tradisional.

---

# 2. Organization Principles

Organisasi AI Engineering mengikuti prinsip:

* Human + AI Collaboration.
* Outcome Driven.
* Platform Mindset.
* Continuous Learning.
* Automation First.
* Ownership.
* Governance Embedded.

# 4. Core Human Roles

---

# 4.1 AI Product Manager

## Responsibility

Menghubungkan:

* business objective,
* user need,
* AI capability.

Tugas:

* Menentukan AI opportunity.
* Membuat product roadmap.
* Mengukur business impact.

---

# 4.2 AI Solution Architect

## Responsibility

Merancang:

* architecture,
* agent ecosystem,
* workflow,
* integration.

Kompetensi:

* Software architecture.
* AI system design.
* Cloud architecture.

---

# 4.3 AI Engineer

## Responsibility

Membangun:

* AI Agent.
* Prompt system.
* AI workflow.
* Evaluation pipeline.

Kompetensi:

* LLM.
* RAG.
* Agent framework.
* Software engineering.

---

# 4.4 Software Engineer

## Responsibility

Membangun:

* application layer,
* backend,
* frontend,
* infrastructure integration.

AI digunakan sebagai engineering multiplier.

---

# 4.5 Agent Engineer

Role baru dalam AI-native organization.

Tanggung jawab:

* Mendesain agent behavior.
* Membuat agent capability.
* Mengoptimalkan agent performance.

---

# 4.6 Knowledge Engineer

Tanggung jawab:

* Knowledge architecture.
* Data preparation.
* Retrieval optimization.

---

# 4.7 AI Evaluation Engineer

Role khusus untuk:

* benchmark,
* quality testing,
* model comparison,
* regression evaluation.

---

# 4.8 AI Operations Engineer

Mengelola:

* AI runtime.
* Model deployment.
* Cost.
* Reliability.

---

# 4.9 AI Governance Engineer

Mengelola:

* policy.
* compliance.
* audit.
* risk.
